MorphoGraphs and Imagery - Practical session 5

Goals
  • Knowing how to apply morphological operators by structuring elements and morphological connected operators to binary and grayscale images thanks to Pink library
  • Understanding the effect of basic morphological operators by structuring elements and of morphological connected operators on binary and grayscale images
  • Knowing how to compose morphological operators to solve some typical problems of binary and grayscale image processing

General Instructions
  • Solve each problem of the session step by step, following the provided guidelines. Note that there can be multiple solutions.
  • For each problem, use the operators (and the programs) given in the associated 'tools' section. These programs must be called from a Linux terminal under the environment settled for the course MorphoGraph and Imagery. Use the provided links to access documentation.
  • For each problem, save in a text file (a shell script) the series of commands that you use.
  • If you do not have time to answer all items during the session, remaining items must be solved as homework after the session. Note that you should send your solution to Problem 3 leading to bonus/malus on your final evaluation of the course.
  • Read the whole description of a problem before starting to look for a solution.

Problem 2: extracting the lobes of cil.pgm

   INPUT: cil.pgm  DESCRIPTION OF THE PROBLEM
     EXPECTED RESULT (2B)
 


The image cil.pgm represents a slice of a cell. In order to know whether this cell is sane or not, it is important to perform a geometrical analysis of the visible bright lobes. In order to perform such analysis, one usually starts by finding the spatial extension of these bright lobes. Problem 2 is about extracting (localizing) the bright lobes of image cil.pgm.
  


Guideline.
  • 2A - Try to extract the "significant" blobs by thresholding (without using a morphological operator)
  • 2B - Extract the "significant" blobs: produce an image like the one above on the right
  • 2C - Same as 2B, but using the operators of the section 'Extra tools.' below
